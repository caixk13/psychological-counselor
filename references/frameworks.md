# 常用评估框架与对话结构

## 概述

这些框架提供**思考结构**，而非必须机械执行的步骤。在实践中应灵活运用，自然融入对话。

## 问题-情绪-认知-行为循环

来访者呈现的问题通常在四个维度之间循环：

```
    触发事件
        ↓
    自动思维（认知层）──→ "我又搞砸了"
        ↓
    情绪反应 ←───────→ 身体感受
    （沮丧、焦虑）      （胸闷、紧绷）
        ↓
    行为反应
    （回避、退缩、发脾气）
        ↓
    后果 → 强化负面认知 → 循环继续
```

**干预切入点可以选任何一环**：
- 认知切入 → CBT 的苏格拉底式提问
- 行为切入 → 行为激活、小步行动
- 情绪切入 → 正念、接纳
- 身体切入 → 呼吸练习、放松训练

## 生物-心理-社会模型（Biopsychosocial Model）

理解来访者的问题时，从三个维度综合考量：

### 生物维度
- 睡眠质量
- 饮食与营养
- 运动情况
- 躯体健康状况
- 家族病史

### 心理维度
- 思维模式（尤其是自动思维）
- 情绪调节能力
- 核心信念
- 应对方式（健康/不健康）
- 自我认知

### 社会维度
- 重要关系（伴侣、家人、朋友）
- 工作/学业环境
- 经济状况
- 社会支持网络
- 文化背景与价值观

**意义**：来访者的问题很少是单一因素导致的。仅关注"心理"可能忽视现实的困难（如经济压力、家庭暴力、工作环境有毒等），导致建议脱离实际。

## 阶段性改变模型（Stages of Change）

Prochaska 和 DiClemente 提出的行为改变螺旋：

```
    前意向期 → 意向期 → 准备期 → 行动期 → 维持期
        ↑                                       ↓
        ←←←←←← 复发（正常！）←←←←←←←←←←←←←←←
```

### 各阶段特征与应对

| 阶段 | 特征 | 合适的对话方向 |
|------|------|---------------|
| **前意向期** | 不认为自己有问题，或没有改变意愿 | 不做说服，只是提供信息，种下种子。尊重、等待 |
| **意向期** | 意识到问题，在纠结是否改变 | 探索矛盾心理（"改变的利弊各是什么？"）。不催促 |
| **准备期** | 决定改变，开始计划 | 帮助制定具体、可操作的小步计划 |
| **行动期** | 正在实施改变 | 支持、鼓励、排除困难、庆祝小胜利 |
| **维持期** | 保持改变成果 | 预防复发策略、建立支持系统 |
| **复发** | 退回到旧行为 | **正常化**（"这是改变过程的一部分"）、分析触发因素、重新开始 |

**关键洞察**：在"前意向期"猛推改变建议 = 无效 + 破坏关系。先搞清楚来访者处于哪个阶段，然后用匹配该阶段的方式对话。

## 基本对话结构

虽然每次对话都是独特的，但以下结构可作为参考框架：

### 第一阶段：建立连接（Opening）
- 开放式邀请："今天想聊什么？"
- 反映初步感受
- 了解背景（不急于深入）
- 建立安全感："你可以按你的节奏来"

### 第二阶段：探索与理解（Exploration）
- 深入了解具体情境
- 引出情绪和认知
- 使用开放式提问和反映
- 识别模式（如果有的话）
- **不要急于跳入"解决"**

### 第三阶段：聚焦与深入（Focusing）
- "我们聊了很多，你觉得哪一点目前最重要？"
- 深入探索选定的焦点
- 评估相关的生物-心理-社会因素
- 如果有需要，读取相关 references 知识

### 第四阶段：干预与合作（Intervention）
- 分享观察和心理学视角
- 提供适合的框架或练习（CBT 思维记录、正念练习、行为激活等）
- 共同探讨可能的行动方案
- 邀请反馈："这个角度对你有帮助吗？还是不太搭？"

### 第五阶段：总结与结束（Closing）
- 总结关键点
- 确认来访者的感受和收获
- 如果有行动计划，确认来访者的承诺程度
- 留下开放的门："以后任何时候想聊都可以来"
- 简短的积极收尾（不虚假乐观）

## 对话中的实时自我监控

在对话过程中，持续问自己：
1. **我在听还是在准备回答？** → 回到倾听
2. **来访者的情绪被我承接住了吗？** → 反映感受
3. **我是不是太急着给建议了？** → 慢下来，回到探索
4. **我有没有越界（诊断、开药、代替专业治疗）？** → 回到能力范围
5. **来访者现在需要什么？**（被理解？被陪伴？具体建议？新视角？）→ 根据需求调整
